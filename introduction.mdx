## What is Microservicios Ecosystem?

This is a reference implementation of a cloud-native microservices architecture built with **Spring Boot 3.4**, **Java 21**, and **Maven Multi-Module**. It demonstrates industry-standard patterns including hexagonal architecture, event-driven communication, and full observability out of the box.

The entire ecosystem can be launched with a single Docker Compose command.

## Architecture at a glance

The ecosystem is composed of six Spring Boot services and a supporting infrastructure layer:

| Service | Port | Role |
|---------|------|------|
| `config-server` | 8888 | Centralized configuration (filesystem) |
| `eureka-server` | 8761 | Service discovery (Netflix Eureka) |
| `auth-service` | 8083 | JWT authentication (BCrypt, H2) |
| `api-gateway` | 8080 | Routing, JWT validation (Spring Cloud Gateway) |
| `inventory-service` | 8082 | Business domain — hexagonal architecture (H2) |
| `notification-service` | 8084 | Event consumer (RabbitMQ or Kafka) |

Supporting infrastructure:

| Component | Port | Role |
|-----------|------|------|
| RabbitMQ | 5672 / 15672 | Message broker option 1 |
| Apache Kafka | 9092 | Message broker option 2 (KRaft) |
| Zipkin | 9411 | Distributed tracing UI |
| Elasticsearch | 9200 | Log storage and search |
| Logstash | 5044 | Log collection and parsing |
| Kibana | 5601 | Log visualization UI |

## Prerequisites

Before you begin, make sure you have the following installed:

- **Java 21** — Required to build and run all Spring Boot services
- **Maven 3.9+** — Multi-module build system
- **Docker Desktop** — Required for the Docker Compose deployment path

<Tip>
  For the Docker path, Docker Desktop must be fully running (not "starting...") before executing any `docker compose` commands.
</Tip>
